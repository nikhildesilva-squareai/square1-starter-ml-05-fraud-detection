# Data dictionary — `transactions.csv`

A card-payment transaction log for fraud detection. **Synthetic — generated by Square 1 AI, free for learners.** ~30,000 rows, one per transaction. **Target: `is_fraud`.**

> ⚠️ Fraud is **rare and imbalanced (~1.5%)** and driven by a *real* relationship
> in the features — your job is to find which ones matter and to score new
> transactions well despite the imbalance. **Accuracy is meaningless here**
> (a model that flags nothing is ~98.5% "accurate"): judge yourself on
> precision, recall, and PR-AUC. Don't peek at the generator — let the model and
> your evaluation tell you the story.
>
> Watch for the dirt: ~4% of rows have a **blank `merchant_category`**,
> `country_code` arrives in **mixed case** (`AU`, `au`, `Us`…), and a few
> `amount` values are extreme **outliers** (data-entry errors).

| Column | Type | Description |
|---|---|---|
| `transaction_id` | string | Identifier, e.g. `T1000042` (drop before modelling). |
| `amount` | float | Transaction amount (AUD). Right-skewed; a few extreme outliers. |
| `hour_of_day` | int | Local hour the transaction was made, 0–23. |
| `country_code` | string | Merchant country. **Mixed case** — normalise before use. |
| `is_foreign` | int | 1 if the transaction is cross-border (country ≠ AU), else 0. |
| `account_age_days` | int | Age of the cardholder's account in days. |
| `txns_last_1h` | int | Spending velocity — transactions on this card in the last hour. |
| `is_new_device` | int | 1 if made from an unrecognised device, else 0. |
| `merchant_category` | string | Spend category (groceries, dining, crypto…). **Blank for ~4% of rows.** |
| `is_fraud` | int | **Target** — 1 if the transaction is fraudulent, else 0. |

**What to do:** clean the dirt (fill missing `merchant_category`, normalise `country_code` case), engineer sensible features, split **stratified** so the rare class is preserved, train a classifier that handles imbalance, and pick an operating threshold from precision/recall — not the default 0.5.
